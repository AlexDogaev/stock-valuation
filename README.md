# Веб-сервис оценки справедливой стоимости акций (MOEX)

Локальный сервис: по эмитентам MOEX считает ожидаемую реальную доходность
(с поправкой на сжатие мультипликатора), накладывает структурный слой,
калибрует под личную инфляцию и hurdle и выдаёт троичный сигнал
**Покупай / Граница / Воздержись**. Математическая модель вшита в бэкенд
(не в Excel), данные котировок подтягиваются с MOEX ISS автоматически.

> ⚠ Не индивидуальная инвестиционная рекомендация. Сервис даёт расчёт и
> суждение для собственного решения пользователя.

## Запуск

```bat
pip install -r requirements.txt
run.bat
```
Открыть http://127.0.0.1:8099

При первом старте БД (`data.sqlite3`) создаётся и засевается из Excel-модели
автоматически. Живые котировки/капа/дивиденды — кнопка «↻ Обновить котировки
MOEX» на скринере (или `POST /api/refresh`).

Полный seed вручную:
```
python -m app.data.seed
```

## Стек

FastAPI + SQLite + Jinja2/vanilla JS. Один процесс, ноль внешней
инфраструктуры. Расчётное ядро — чистые функции на Python (`app/core/`),
покрытые тестами; от веб-слоя не зависит.

```
app/
  core/        расчётное ядро (чистые функции + тесты)
    valuation.py   зрелый P/B, полная доходность, троичный сигнал, тест r−g
    structural.py  6 структурных измерений → множитель к g
    inflation.py   личный дефлятор (тактич./стратег.)
    rate.py        требуемая доходность r из траектории КС
    classify.py    зрелый/растущий/вырождающийся + фаза N
    growth.py      двухстадийный DCF (калиброванный + проекция)
    sotp.py        Ozon Sum-of-the-Parts
    leverage.py    ROIC, покрытие, выживание
    body_trend.py  тренд тела (прокси из выручки/капекса/ROIC)
    barbell.py     барбелл-калькулятор
    portfolio.py   факторное разложение, концентрация, стресс
    backtest.py    проверка предсказательности
    engine.py      сборка: данные БД → ядро → карточка
  data/        MOEX-клиент, схема SQLite, seed из Excel
  web/         API-роуты, шаблоны, статика
tests/         pytest — согласованность ядра с числами Excel
```

## Тесты

```
python -m pytest -q
```
Тесты заземляют ядро на якорные числа Excel-модели (Сбер, X5, Яндекс, Ozon,
Лукойл). Совпадение = внутренняя согласованность модели (НЕ предсказательность —
её доказывает только backtest на истории, лист `Backtest`).

## API (основное)

| Метод | Путь | Назначение |
|------|------|-----------|
| GET  | `/api/issuers` | список с сигналом под текущие настройки |
| GET  | `/api/issuers/{secid}` | карточка: метрики (с источником), слои, сигнал |
| GET/PUT | `/api/settings` | hurdle, буфер, режим, премия, дефлятор |
| PUT  | `/api/issuers/{secid}/structural` | экспертные структурные баллы |
| GET/POST | `/api/portfolio` | состав + факторы + концентрация + стресс |
| POST | `/api/refresh` | обновить котировки/капу/дивиденды с MOEX |
| GET  | `/api/backtest?years=1,2,3` | прогон модели на истории MOEX |
| POST | `/api/calc/{barbell,maturity,growth_calibrated,sotp,leverage,body_trend,inflation}` | калькуляторы |

## Backtest (проверка предсказательности)

Страница `/backtest` прогоняет модель на истории MOEX: realized — фактический
номинальный CAGR (цена тогда → сегодня + дивиденды), predicted — implied
доходность модели на исторической цене. **Допущение:** predicted считается с
ТЕКУЩИМ фундаменталом (архива отчётности нет) → результат смещён и показывает,
что модель в текущем виде НЕ предсказывает (ср. ошибка ~15 п.п.). Это честный
вывод, согласный с методологией: «согласованность ≠ предсказание». Настоящий
backtest требует исторического фундаментала на дату входа.

## Автоматизация уровня 2 — T-Invest API (подключено)

Фундаментал (ROE, чистая прибыль, payout, капитал, ROIC) подтягивается из
**T-Invest API** (`GetAssetFundamentals`), дополняя seed из Excel. `g_base`,
`compression`, `wacc`, `body_trend` остаются модельными.

**Токен** (нужен брокерский счёт Т-Банк → Настройки → токены Invest API):
```bat
set TINVEST_TOKEN=ваш_токен
```
или положить токен в файл `.tinvest_token` в корне проекта (в `.gitignore`).

Обновить: кнопка «↻ Обновить фундаментал из T-Invest» на странице Настройки,
либо `POST /api/refresh_fundamentals`. Без токена — graceful skip с подсказкой.

> Сверка при первом запуске: ROE Сбера должен стать ≈ 0.227 (не 22.7) — нормализация
> процентов в `norm_pct`. Если T-Invest отдаёт доли, а не проценты — значения уже
> корректны. API даёт текущий срез, истории по годам нет.

## Автоматизация данных уровня 2 — другие провайдеры

MOEX ISS не отдаёт МСФО (прибыль/ROE/капитал/payout) — узкое место (SPEC §3.2).
По итогам исследования провайдеров (июнь 2026):
- **T-Invest API (Тинькофф/Т-Банк), метод `GetAssetFundamentals`** — лучший
  бесплатный путь: REST/gRPC + SDK, отдаёт net_income, ROE, ROA, ROIC, P/B,
  `dividend_payout_ratio_fy`, капитализацию по MOEX. Нужен токен (счёт брокера).
  Ограничение: только текущий срез, без истории по годам (накапливать снапшотами).
- **Недорогой полуавтомат:** T-Invest + FinanceMarker.ru API (~499–990 ₽/мес,
  лимит ~400 запросов/день; коммерческую лицензию уточнять).
- **Прод/эталон:** RUData (Интерфакс+НРД) / СПАРК (от ~200 тыс ₽/год) /
  MOEX Datashop (от ~$5000/мес) — корпоративный бюджет.
- **e-disclosure.ru** — первоисточник, но в основном PDF (XBRL только у банков/НФО);
  парсер дорог в разработке. Зарубежные провайдеры — покрытие РФ после 2022 не
  подтверждено, закладываться нельзя.

Абстракция изолирована в таблице `financials` — источник меняется без трогания ядра.

## Известные ограничения (по дизайну)

- **Данные уровня 2** (ROE, прибыль, капитал, payout) — ручной ввод / seed из
  Excel по ~25 ликвидным именам. Автоматизация — см. раздел выше.
- **Дивдоходность TTM** помечается флагом ⚠ при аномалии (разовый спецдив:
  TTM > 20% или кратно выше исторической нормы) — на скринере и в карточке.
- **Дивдоходность = TTM** (сумма выплат за 12 мес / цена, SPEC §4.1). Разовые
  спецдивиденды (напр. X5 после редомициляции) завышают её → требуют ручной
  проверки, как и ресурсные имена (SPEC §9). На скринере такие имена видны по
  аномально высокой дивдоходности.
- **Факторные беты и корреляции** — качественный каркас (загрузки по сектору).
  Точные числа — регрессия исторических доходностей MOEX (фаза 3, не в MVP).
- **Структурные баллы** — экспертное суждение, не из API. Правятся в карточке
  эмитента (раздел «Структурные баллы»).
